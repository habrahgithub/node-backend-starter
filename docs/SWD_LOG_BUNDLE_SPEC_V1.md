# SWD Log Bundle Spec v1

- Version: `1.0`
- Status: canonical contract (Phase 1 executable in `swd-vault ingest logs`)
- Scope: SWD governance logs converted to NDJSON and ingested via Vault CLI only

## Goals

- Define essential governance artifacts for SWD operations history.
- Convert logs to deterministic NDJSON records (stable hashing).
- Ingest append-only through `swd-vault` (no direct DB writes).
- Preserve provenance and dedupe guarantees.

## NDJSON Record Contract

One JSON object per line:

```json
{
  "bundle_id": "swd-log-bundle-2026-02-20",
  "bundle_created_at": "2026-02-20T12:00:00Z",
  "source_path": "projects/docsmith-payment-gateway/SWD-Execution-Log.md",
  "source_type": "swd_execution_log",
  "record_kind": "entry",
  "record_ts": "2026-02-19T18:42:10Z",
  "title": "Payment webhook retry policy tightened",
  "body_excerpt": "Retry window set to 24h with deterministic backoff...",
  "tags": ["payment", "governance", "ops"],
  "content_sha256": "abc123...",
  "record_fingerprint": "def456...",
  "provenance": {
    "run_profile": "manual",
    "cli_version": "vault-cli@1.0.0+a1b2c3d",
    "host_fingerprint": "8f2dbe13..."
  }
}
```

Required fields:

- `bundle_id`
- `source_path`
- `source_type`
- `record_kind`
- `record_ts` (best effort; fallback to file mtime in UTC)
- `content_sha256`
- `record_fingerprint`

## Canonicalization Rules (Hash Stability)

Apply before hash/fingerprint:

1. UTF-8 decode with replacement for invalid bytes.
2. Normalize line endings to `\n`.
3. Trim trailing whitespace on each line.
4. Collapse 3+ blank lines to 2.
5. Strip volatile export markers (timestamps generated by tooling).
6. Normalize Unicode to NFC.
7. `body_excerpt` max length: `500` chars after redaction.

`content_sha256` input:

- Canonicalized record body text.

`record_fingerprint` input:

- `bundle_id + "|" + source_path + "|" + record_kind + "|" + content_sha256`

## Ingestion Mapping to Vault Events

- `source_type=swd_execution_log` -> event `type=ops_sweep` or `type=note` by parser rule.
- `source_type=notion_execution_log` -> event `type=decision` or `type=note`.
- `source_type=decisions` -> event `type=decision`.
- `source_type=swd_work_order_update` -> event `type=task` or `type=decision` (tag-driven).
- `source_type=swd_daily_brief` -> event `type=note`.

Event requirements:

- Respect current hash chain insertion flow.
- Preserve `details_json` provenance and source metadata.
- Use pre-insert duplicate checks; skip duplicates silently, count in summary.

## Dedupe Strategy

Three levels:

1. Bundle duplicate:
   - `bundle_fingerprint = sha256(sorted(source_path + content_sha256))`
   - If previously ingested, skip full bundle.
2. File-level duplicate:
   - `file_fingerprint = sha256(source_path + file_content_sha256)`
3. Record-level duplicate:
   - `record_fingerprint` as defined above.

No-op rule:

- If all records are duplicates, emit no ingest-summary event (consistent with existing no-op suppression).

## Safety Constraints

- No direct writes to `vault/db/vault.sqlite`.
- No raw secret capture; redact token-like strings and `.env` style assignments.
- Do not store full file bodies in DB by default; store metadata + headings + excerpt.
- Always run `./swd-vault verify chain` after log ingest batch.

## Output Metrics Contract

Minimum summary metrics:

- files_scanned
- records_scanned
- records_inserted
- records_skipped
- duplicate_ratio
- bundle_fingerprint
- duration_ms

These metrics should map into War Room ingest coverage and anomaly badges.
