# Context Execution

## Workflow

1. Locate canonical docs and target project entry points.
2. Identify changed files, dependency boundaries, and runtime assumptions.
3. Define success criteria and verification commands before editing.
4. Capture blockers, missing inputs, and risk areas.
5. Keep context concise and path-based for fast execution.

## Focus Areas

- Which project/module is in scope.
- Which commands validate completion.
- Which environments/secrets are required.
- Which existing skills should be combined.

## Output Rules

- Provide a short execution map: scope, risks, verify commands.
- Prefer file references over long logs.
- Update context when scope shifts.
