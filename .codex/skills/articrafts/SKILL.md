# Articrafts

## Workflow

1. Identify required artifacts for the task outcome.
2. Separate durable artifacts from temporary/generated noise.
3. Store artifacts in consistent, predictable locations.
4. Ensure artifact naming includes purpose and date/version.
5. Verify references to artifacts in logs or summaries.

## Artifact Types

- Checkup reports and test summaries.
- Screenshots and evidence captures.
- Generated bundles required for release validation.
- One-off diagnostics that should not be committed.

## Output Rules

- List artifact path, purpose, and retention expectation.
- Flag artifacts that should be ignored or cleaned.
- Keep artifact handling aligned with storage policy.
