# Context

## Workflow

1. Read `docs/CONTEXT.md` first.
2. Read `docs/DECISIONS.md` and the target project `README.md` next.
3. Locate only the files directly related to the request.
4. Summarize assumptions, constraints, and affected paths before large edits.
5. Keep context lean by citing file paths instead of pasting long logs.

## Guardrails

- Prefer canonical documents over ad-hoc notes.
- Keep summaries short, concrete, and path-based.
- Re-check repo context when switching between projects in `projects/`.
